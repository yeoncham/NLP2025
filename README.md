# 명예훼손 데이터 기반 다중 클래스 분류 프로젝트

> **현재 진행 상태:** 데이터 전처리 완료  
> **팀 구성:** 곽건 · 김종민 · 위찬우 · 최재혁 · 최호준  
> **과목:** 자연언어처리

---

## 프로젝트 개요

### 문제 정의
- **온라인 명예훼손 사건의 증가**로 인한 사회적 이슈
- **법률 종사자**들의 업무 시간 절감 필요
- **일반인**이 법적 판단을 미리 예측할 수 있도록 지원
- **판단 기준의 일관성** 확보

### 프로젝트 목표
> "명예훼손으로 추정되는 문장을 입력하면,  
> 모델이 판결 결과(유죄 / 무죄 / 기타)를 예측하고  
> 그 이유를 함께 제시하는 시스템 구축"

---

## 프로젝트 파이프라인

1. **데이터 수집 및 준비**
2. **데이터 전처리**
3. **모델 설계 및 학습**
4. **모델 평가 및 분석**
5. **데이터 증강**
6. **UI 및 시스템 구현**

---

## 데이터 전처리

### EDA 및 정제
- NULL 값 및 데이터 타입 검사
- 불필요한 열 제거 및 값 통일
- 텍스트 정규화 수행

### 라벨링(정답 태깅)
- **판결문 `[주문]`** 부분에서 `무죄`, `징역`, `벌금` 등의 키워드를 탐색하여 자동 라벨링
- **“상고를 기각한다”** 등의 케이스는 `[이유]` 부분을 참고하여 **수작업 라벨링** 수행

### 데이터 누수 방지
- `"위반"`, `"추행"` 등 **결과를 암시하는 단어** 제거
- 학습 과정에서 결과 정보가 직접 노출되지 않도록 정제

---

## 모델 설계

| 구성 요소 | 설명 |
|------------|------|
| **Text Embedding + MLP** | 오픈소스 임베딩 모델 사용하여 임베딩 모델 뒤의 MLP 학습 |
| **Fine-tuning** | LoRA 또는 특정 Layer만 학습하여 비용 절감 |
| **Ontology Network (보완 모델)** | 인물 간 관계를 그래프로 표현하여 규칙 기반 판단 보조 |

---

## 데이터 증강 계획

| 기법 | 설명 |
|------|------|
| **LLM 기반 합성 데이터 생성** | GPT 등 LLM을 이용한 판례 유사 텍스트 생성 |
| **텍스트 변형 기법** | SR(유의어 교체), RD(단어 삭제) |
| **역번역(Back-translation)** | 의미 유지 + 표현 다양성 확보 |
| **법률 논리 재조합** | 동일한 논리를 가진 사건을 재구성하여 새로운 데이터 생성 |
